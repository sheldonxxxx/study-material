# Dependencies

## Overview

The agency-agents repository has **zero external package dependencies**. It is a pure content repository using only:
- Markdown (text files)
- YAML (frontmatter in Markdown)
- Bash (shell scripts)

No npm, pip, bundlers, or runtime dependencies are required.

## Dependency Analysis

### Runtime Dependencies

| Component | None Required | Notes |
|-----------|--------------|-------|
| Package Manager | None | Not a Node.js/Python project |
| Framework | None | Pure Markdown content |
| Runtime | None | No JavaScript/TypeScript runtime |
| Build System | None | Scripts run directly |

### Development Tooling

| Tool | Required | Used For |
|------|----------|----------|
| Git | Yes | Version control (user responsibility) |
| Bash | Yes | Script execution |
| GitHub Actions | No | CI/CD (automated by GitHub) |

### System Requirements

- Unix-like operating system (Linux, macOS, WSL)
- Bash 4.0+
- Standard Unix utilities: `awk`, `sed`, `grep`, `head`, `find`, `sort`, `xargs`

### Portable Script Design

The shell scripts use portable, POSIX-compliant constructs:

```bash
# Uses portable shebang
#!/usr/bin/env bash

# Uses portable command substitutions
$(command) rather than `command`

# Uses set for strict mode
set -euo pipefail

# Avoids GNU-specific extensions
```

## Script Dependencies

### convert.sh

Uses only standard utilities:
- `awk` — Text processing, field extraction
- `sed` — Text transformation, regex
- `grep` — Pattern matching
- `head` — First line extraction
- `find` — File discovery
- `sort` — Sorting
- `xargs` — Parallel execution
- `mkdir` — Directory creation
- `cp` — File copying
- `mktemp` — Temporary files

### install.sh

Uses only standard utilities:
- `bash` — Script interpreter
- `mkdir` — Directory creation
- `cp` — File copying
- `rm` — Cleanup
- `read` — User input (interactive mode)
- Standard shell built-ins

### lint-agents.sh

Uses only standard utilities:
- `awk` — Frontmatter extraction, validation
- `sed` — Text processing
- `grep` — Pattern matching
- `head` — First line check
- `wc` — Word count
- Standard shell built-ins

## Gitignore

Generated files are explicitly excluded:

```
_site/
*.mdc
CONVENTIONS.md
.windsurfrules
```

These are generated by `convert.sh` and installed by `install.sh` but never committed.

## External Services (Agent-Declared)

Agents may declare external service dependencies:

```yaml
services:
  - name: Service Name
    url: https://service-url.com
    tier: free
```

These are **agent capabilities**, not project dependencies. The repository itself does not depend on any external services.

## No Lock Files

There are no lock files because there are no package dependencies:
- No `package.json`
- No `package-lock.json`
- No `yarn.lock`
- No `Pipfile.lock`
- No `requirements.txt`

## Dependency-free Benefits

1. **Zero installation** — Clone and use immediately
2. **No version conflicts** — Nothing to upgrade
3. **Security surface** — No third-party code to audit
4. **Offline capable** — Works without network access
5. **Fast CI** — No `npm install` or dependency caching needed
6. **Portable** — Works on any Unix-like system
