# Community

## Overview

The agency-agents project is a **community-driven collection of AI agent personalities** that originated from a Reddit thread and has grown into a multi-tool, multi-language project with active participation from developers worldwide.

## Project Origin

The project began from a Reddit thread on r/ClaudeAI about AI agent specialization. After months of iteration from real-world usage, it evolved into a comprehensive collection of 144+ specialized agents across 12 divisions.

## Contribution Pathways

### 1. Create a New Agent

The fastest path to contribution — one markdown file.

Process:
1. Fork the repository
2. Choose appropriate category directory
3. Create agent file following template
4. Test in real scenarios
5. Submit PR with agent information

### 2. Improve Existing Agents

- Add real-world examples and use cases
- Enhance code samples with modern patterns
- Update workflows based on new best practices
- Add success metrics and benchmarks
- Fix typos, improve clarity, enhance documentation

### 3. Share Success Stories

- Post in GitHub Discussions
- Add case study to README
- Write blog post and link it
- Create video tutorial

### 4. Report Issues

- Check if issue already exists
- Provide clear reproduction steps
- Include context about use case
- Suggest potential solutions

## Community Guidelines

### Code of Conduct

From CONTRIBUTING.md:
- **Be Respectful** — Treat everyone with respect, healthy debate encouraged, no personal attacks
- **Be Inclusive** — Welcome people of all backgrounds and identities
- **Be Collaborative** — What we create together is better
- **Be Professional** — Keep discussions focused on improving agents and community

### PR Philosophy

The maintainers emphasize:
- **One markdown file is the sweet spot** — Fastest path to merged PR
- **Start a Discussion first** — For new tooling, build systems, CI workflows, architectural changes
- **Never check in generated files** — `_site/`, compiled assets, converted agent files
