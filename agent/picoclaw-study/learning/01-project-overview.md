# PicoClaw Project Overview

## Project Purpose

**PicoClaw** is an ultra-lightweight personal AI assistant written entirely in Go. It was initiated by [Sipeed](https://sipeed.com) as an open-source project, rebuilt from scratch inspired by NanoBot. The project emphasizes extreme resource efficiency, targeting operation on hardware costing as little as $10 with memory footprints under 10MB.

## Core Architecture

### Design Philosophy
- **Ultra-lightweight**: Core memory footprint <10MB (99% smaller than OpenClaw)
- **Cross-platform**: Single binary runs on x86_64, ARM64, MIPS, RISC-V, and LoongArch
- **AI-bootstrapped**: 95% of core code was generated by an AI Agent with human-in-the-loop review
- **Fast boot**: Boots in under 1 second on 0.6GHz single-core processors

### Key Components

#### 1. Agent Core (`pkg/agent/`)
The agent package implements the core AI agent loop with:
- **Event bus**: Pub/sub system for component communication
- **Hook system**: Extensible hooks for intercepting and modifying agent behavior
- **Memory management**: Context budget tracking and caching
- **MCP integration**: Native Model Context Protocol support
- **SubTurn**: Sub-agent coordination and lifecycle management
- **Thinking**: Thought process management

#### 2. Authentication (`pkg/auth/`)
Secure credential management with:
- OAuth 2.0 flows (browser, device code, setup-token)
- PKCE (Proof Key for Code Exchange) support
- Secure credential store with atomic file operations
- Token refresh handling

#### 3. Channels (`pkg/channels/`)
Integration with 17+ messaging platforms:
- **IM Platforms**: Telegram, Discord, WhatsApp, WeChat, QQ, Slack, LINE
- **Enterprise**: Feishu/Lark, DingTalk, WeCom, Matrix/IRC
- **Specialized**: OneBot protocol, MaixCam, Pico native protocol

#### 4. Providers (`pkg/providers/`)
Support for 30+ LLM providers through unified interface:
- **Cloud**: OpenAI, Anthropic, Google Gemini, DeepSeek, Zhipu, Qwen, Groq, etc.
- **Local**: Ollama, vLLM, LM Studio
- **Enterprise**: AWS Bedrock, Azure OpenAI, GitHub Copilot

### Configuration System

PicoClaw uses a two-file configuration approach:
- **`config.json`**: Non-sensitive configuration (model list, channel settings, agent defaults)
- **`.security.yml`**: Sensitive data (API keys, tokens, secrets) with environment variable override support

### Build System

- **Go 1.25+** required
- **Makefile targets**: `make build`, `make build-launcher`, `make build-all`, `make build-pi-zero`
- **Docker support**: `docker compose` with launcher profile
- **Multiple build tags**: `bedrock` tag for AWS Bedrock support

### CLI Commands

| Command | Description |
|---------|-------------|
| `picoclaw onboard` | Initialize config and workspace |
| `picoclaw agent -m "..."` | One-shot question |
| `picoclaw agent` | Interactive chat mode |
| `picoclaw gateway` | Start gateway for chat integration |
| `picoclaw auth login` | Authenticate with providers |
| `picoclaw cron list/add/remove` | Manage scheduled tasks |
| `picoclaw skills install/list` | Manage skills |
| `picoclaw migrate` | Migrate from older versions |

### Launchers

1. **WebUI Launcher** (`picoclaw-launcher`): Browser-based configuration UI at `http://localhost:18800`
2. **TUI Launcher** (`picoclaw-launcher-tui`): Terminal UI for headless/server environments

2. **Security Hardening**: Prompt injection defense, sandboxing, OAuth 2.0 migration
3. **Connectivity**: Protocol-based architecture, local model deep integration, skill marketplace
4. **Advanced Capabilities**: MCP support, browser automation, multi-agent collaboration
