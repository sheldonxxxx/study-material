## Contribution Model

### How to Contribute

1. **Fork** the repository
2. **Clone** your fork
3. **Create a feature branch**: `git checkout -b feat/my-feature`
4. **Make changes** in `src/`
5. **Build and test**: `npm run build && npm test`
6. **Commit** with clear message
7. **Push and open PR**

### Contribution Guidelines

From `CONTRIBUTING.md`:

| Guideline | Details |
|-----------|---------|
| **Language** | TypeScript (strict mode) |
| **Module System** | ESM (`"type": "module"`) |
| **Philosophy** | Keep it simple, avoid over-engineering |
| **Testing** | Add/update tests in `test/` |
| **PR Scope** | One concern per PR |
| **Commit Style** | Descriptive: `Add X`, `Fix Y`, `Update Z` |

### Code of Conduct

> "Be respectful and constructive. We're all here to build something useful together."

## Growth Opportunities

### Strengths

1. **Clear vision**: Git-native agent paradigm is novel
2. **Minimal dependencies**: Easy to understand and maintain
3. **MIT license**: Permissive for adoption
4. **TypeScript**: Good developer experience

### Challenges

1. **Single maintainer**: Sustainability risk
2. **Documentation gaps**: API reference, troubleshooting
3. **No CI for tests**: Quality gate missing before publish
4. **Limited community artifacts**: No showcases, examples gallery

### Opportunities

1. **Expand core team**: Add collaborators with commit access
2. **Build plugin ecosystem**: Showcase community plugins
3. **Enterprise features**: Already has compliance/audit, could market
4. **Video content**: Demo the git-native workflow
