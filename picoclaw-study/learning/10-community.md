# Community

## Project Origin

PicoClaw is an **independent open-source project initiated by [Sipeed](https://sipeed.com)** (a hardware company known for RISC-V boards like the LicheeRV). It was written entirely in Go from scratch, not as a fork of NanoBot, OpenClaw, or any other existing project.

The project was bootstrapped through a "self-bootstrapping" process — the AI Agent itself drove the architecture migration and code optimization from an earlier prototype to pure Go. Approximately 95% of the core code was generated by an AI Agent and refined through human-in-the-loop review.

## Contributing

See `CONTRIBUTING.md` for full guidelines. Key points:

### Ways to Contribute
- Bug reports (use issue template)
- Feature requests (use issue template; discuss before implementing)
- Code (fixes, features; follow workflow below)
- Documentation (README improvements, translations, inline comments)
- Testing (new hardware, channels, LLM providers)

### Branch Strategy
- **`main`** — Active development; protected (PRs only, no direct pushes)
- **`release/x.y`** — Stable releases; more strictly protected than `main`
- Contributors branch off `main` and target `main` in PRs

### Merge Requirements
1. CI passes (lint, vuln check, tests)
2. At least one maintainer approval
3. No unresolved review comments
4. Complete PR template (including AI disclosure section)

### AI-Assisted Contributions

The project explicitly embraces AI-assisted development. Every PR must disclose AI involvement:

| Level | Description |
|-------|-------------|
| Fully AI-generated | AI wrote the code; contributor reviewed and validated |
| Mostly AI-generated | AI produced draft; contributor made significant modifications |
| Mostly Human-written | Contributor led; AI provided suggestions or none |

**Responsibilities**: Contributors must read, understand, and test all AI-generated code. Security review is especially emphasized for AI-generated tool implementations and channel handlers.

### Commit Style
- Conventional Commits format (e.g., `feat: Add Ollama provider (#491)`)
- Imperative mood: "Add retry logic" not "Added retry logic"
- Reference related issues: `Fix session leak (#123)`
- Squash merge to `main` for clean history

## Hardware Compatibility Testing

Community members are encouraged to test PicoClaw on various hardware and report results. A [hardware compatibility list](docs/hardware-compatibility.md) documents tested boards including:
- RISC-V boards (LicheeRV-Nano, Milk-V)
- Raspberry Pi models
- x86 boards
- Android phones (via Termux)
- And more

Boards not on the list can be submitted via PR.
