## Contribution Process

1. Fork the repository
2. Create descriptive branch (`feature/your-feature-name`)
3. Add tests for all new features
4. Submit PR with description referencing related issues
5. Ensure linting and tests pass locally

### Contribution Areas
- Writing tests (high-priority need)
- Improving documentation
- Adding new swarm architectures
- Enhancing agent capabilities
- Removing defunct code

### Coding Standards
- Mandatory type annotations for all functions/methods
- Docstrings: Google Python Style Guide or NumPy Docstring Standard
- Required unit tests for all features and bug fixes
- PEP 8 compliance via flake8, black, pylint

## GitHub Infrastructure

### Issue Templates
- Bug Report: Auto-assigns to kye@swarms.world, auto-labels with `bug`
